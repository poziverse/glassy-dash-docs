## Design Principles

1. **User-First**: Write for the user, not yourself
2. **Clear & Concise**: Simple language, active voice
3. **Actionable**: Every section should have a clear purpose
4. **Searchable**: Use consistent terminology
5. **Maintainable**: Modular structure, easy to update
6. **Accessible**: Follow WCAG 2.1 AA guidelines
7. **Modern**: Follow 2026 documentation trends
8. **Visual**: Use diagrams, screenshots, and videos
